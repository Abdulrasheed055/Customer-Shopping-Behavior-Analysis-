````markdown
# 🛠 Customer Shopping Behavior Analysis - SQL Insights

This section documents all SQL queries used to analyze customer shopping behavior from **3,900 transactions**. Each query includes its **purpose, dataset, and insight**.

## 1️⃣ Revenue by Gender
**Query Purpose:** Compare total revenue generated by male vs female customers.  

**Insight:**
- Identify which gender contributes more to revenue.  
- Helps guide gender-targeted marketing campaigns.

```sql
SELECT gender, 
       SUM(purchase_amount) AS total_revenue
FROM customer_transactions
GROUP BY gender;
````

## 2️⃣ High-Spending Discount Users

**Query Purpose:** Identify customers who used discounts but still spent above the average purchase amount.

**Insight:**

* Reveals high-value customers who respond to promotions.
* Supports targeted promotional strategies.

```sql
SELECT customer_id, 
       SUM(purchase_amount) AS total_spent
FROM customer_transactions
WHERE discount_applied = TRUE
GROUP BY customer_id
HAVING SUM(purchase_amount) > (SELECT AVG(purchase_amount) FROM customer_transactions);
```

---

## 3️⃣ Top 5 Products by Average Rating

**Query Purpose:** Find products with the highest review ratings.

**Insight:**

* Determines top-quality products based on customer feedback.
* Guides inventory, marketing, and upsell focus.

```sql
SELECT item_purchased, 
       AVG(review_rating) AS avg_rating
FROM customer_transactions
GROUP BY item_purchased
ORDER BY avg_rating DESC
LIMIT 5;
```

## 5️⃣ Subscribers vs Non-Subscribers

**Query Purpose:** Compare average spend and total revenue across subscription status.

**Insight:**


## 6️⃣ Discount-Dependent Products

**Query Purpose:** Identify products with the highest percentage of discounted purchases.

**Insight:**

* Highlights products that are sensitive to price promotions.
* Guides discount campaigns and pricing strategy.

```sql
SELECT item_purchased,
       COUNT(CASE WHEN discount_applied = TRUE THEN 1 END) * 100.0 / COUNT(*) AS discount_percentage
FROM customer_transactions
GROUP BY item_purchased
ORDER BY discount_percentage DESC
LIMIT 5;
```

## 7️⃣ Customer Segmentation

**Query Purpose:** Classify customers into New, Returning, and Loyal based on purchase history.

**Insight:**

* Provides customer segmentation for marketing and loyalty programs.
* Helps identify retention and engagement opportunities.

```sql
SELECT customer_id,
       CASE 
           WHEN COUNT(*) = 1 THEN 'New'
           WHEN COUNT(*) BETWEEN 2 AND 5 THEN 'Returning'
           ELSE 'Loyal'
       END AS segment
FROM customer_transactions
GROUP BY customer_id;
```

## 8️⃣ Top 3 Products per Category

**Query Purpose:** List the most purchased products within each category.

**Insight:**

* Determines best-selling products per category.
* Guides inventory, promotions, and merchandising strategy.

```sql
WITH ranked_products AS (
    SELECT category, 
           item_purchased,
           COUNT(*) AS purchase_count,
           ROW_NUMBER() OVER(PARTITION BY category ORDER BY COUNT(*) DESC) AS rank
    FROM customer_transactions
    GROUP BY category, item_purchased
)
SELECT category, item_purchased, purchase_count
FROM ranked_products
WHERE rank <= 3
ORDER BY category, purchase_count DESC;
```

## 🔟 Repeat Buyers & Subscriptions

**Query Purpose:** Check if customers with more than 5 purchases are more likely to subscribe.

**Insight:**

* Understand correlation between repeat purchase behavior and subscriptions.
* Guides retention and loyalty programs.

```sql
SELECT customer_id,
       COUNT(*) AS total_purchases,
       MAX(subscription_status) AS subscription_status
FROM customer_transactions
GROUP BY customer_id
HAVING COUNT(*) > 5;
```
